# 问题一 日用电量、新能源发电量、网购电量、上网电量 — 计算式详解

---

## 一、前置定义

在逐时层面（$t = 0, 1, \ldots, 23$），系统的核心变量来自 `config.py` 和附件数据：

| 变量 | 含义 | 单位 | 来源 |
|------|------|:--:|------|
| $P_{\text{wind}}(t)$ | 风电出力 | MW | `config.get_typical_wind()` = 附件2风电标幺值 × 40 MW |
| $P_{\text{solar}}(t)$ | 光伏出力 | MW | `config.get_typical_solar()` = 附件2光伏标幺值 × 64 MW |
| $P_{\text{load}}(t)$ | 常规电负荷 | MW | `config.get_load_profile()` = 附件1标幺值 × 6 MW |
| $P_{\text{eq}}$ | 设备总功率（**常数**） | MW | `ALKEL_RATED(10) + PEMEL_RATED(10) + NH3_RATED(0.75) = 20.75` |

---

## 二、逐时功率平衡与购售电决策

功率平衡方程为：

$$\boxed{P_{\text{wind}}(t) + P_{\text{solar}}(t) + P_{\text{buy}}(t) = P_{\text{sell}}(t) + P_{\text{eq}} + P_{\text{load}}(t)}$$

定义**净功率**（风光出力扣除园区全部用电后剩余量）：

$$\boxed{P_{\text{net}}(t) = P_{\text{wind}}(t) + P_{\text{solar}}(t) - P_{\text{eq}} - P_{\text{load}}(t)}$$

购售电由 $P_{\text{net}}(t)$ 的符号唯一决定（零自由度，纯解析）：

$$\boxed{P_{\text{buy}}(t) = \max\big(-P_{\text{net}}(t),\ 0\big), \qquad P_{\text{sell}}(t) = \max\big(P_{\text{net}}(t),\ 0\big)}$$

> **物理含义**：
> - $P_{\text{net}}(t) < 0$ → 风光不够用 → 差额 $|P_{\text{net}}(t)|$ 从电网购入
> - $P_{\text{net}}(t) > 0$ → 风光有盈余 → 多余电力卖回电网
> - 二者**天然互斥**，同一时刻 $\text{buy}$ 和 $\text{sell}$ 不可能同时非零

对应代码（`q1_baseline.py` 第46–54行）：

```python
for t in range(n_hours):
    net_power = wind[t] + solar[t] - TOTAL_EQ_POWER - load[t]
    if net_power >= 0:
        sell[t] = net_power
        buy[t] = 0.0
    else:
        buy[t] = -net_power
        sell[t] = 0.0
```

## 三、四个日电量指标的精确计算式

### 3.1 新能源发电量 $E_{\text{RE}}$（日总）

$$\boxed{E_{\text{RE}} = \sum_{t=0}^{23} \Big[P_{\text{wind}}(t) + P_{\text{solar}}(t)\Big] \times 1000 \quad (\text{kWh})}$$

**代码对应**（第58–61行）：

```python
wind_kwh  = float(np.sum(wind))  * 1000   # Σ P_wind(t) × 1000
solar_kwh = float(np.sum(solar)) * 1000   # Σ P_solar(t) × 1000
total_gen  = wind_kwh + solar_kwh          # E_RE
```

> **为何 ×1000**：$P_{\text{wind}}(t)$ 单位为 MW，乘以 1 小时得 MWh，再 ×1000 转换为 kWh。本质上：
> $$E_{\text{wind}} = \sum_t P_{\text{wind}}(t) \cdot \Delta t = \sum_t P_{\text{wind}}(t)\ [\text{MW}] \times 1\ [\text{h}] \times 1000\ [\text{kWh/MWh}]$$

**典型日数值**：$761.76\ \text{MWh}$（= 761,760 kWh）

---

### 3.2 日用电量 $E_{\text{load}}^{\text{total}}$（园区总用电）

$$\boxed{E_{\text{load}}^{\text{total}} = \sum_{t=0}^{23} \Big[P_{\text{eq}} + P_{\text{load}}(t)\Big] \times 1000 = \underbrace{P_{\text{eq}} \times 24 \times 1000}_{\text{设备用电}} + \underbrace{\sum_{t=0}^{23} P_{\text{load}}(t) \times 1000}_{\text{常规负荷用电}} \quad (\text{kWh})}$$

**代码对应**（第62行）：

```python
total_load = load_kwh + (TOTAL_EQ_POWER * 1000 * n_hours)
#                        ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
#                        20.75 × 1000 × 24 = 498,000 kWh
```

其中 $\text{load\_kwh} = \sum_t P_{\text{load}}(t) \times 1000$。

拆解来看：

$$\begin{aligned} E_{\text{load}}^{\text{total}} &= \underbrace{20.75 \times 24 \times 1000}_{= 498,000\ \text{kWh (设备)}} + \underbrace{\sum_{t=0}^{23} P_{\text{load}}(t) \times 1000}_{\text{常规负荷}} \\ &\approx 498,000 + (\text{负荷积分}) \approx 498,000\ \text{kWh} \end{aligned}$$

> 由于常规负荷（峰值6 MW）相对设备用电（20.75 MW恒定）较小，总用电量约为 **498 MWh**。

---

### 3.3 网购电量 $E_{\text{buy}}$

$$\boxed{E_{\text{buy}} = \sum_{t=0}^{23} P_{\text{buy}}(t) \times 1000 = \sum_{t=0}^{23} \max\big(-P_{\text{net}}(t),\ 0\big) \times 1000 \quad (\text{kWh})}$$

**代码对应**（第59行）：

```python
buy_kwh = float(np.sum(buy)) * 1000
```

其中 `buy[t]` 在每一时刻由 $P_{\text{net}}(t) < 0$ 时取 $-P_{\text{net}}(t)$，否则为 0。

**等价写法**（用指示函数 $\mathbb{I}$）：

$$E_{\text{buy}} = \sum_{t: P_{\text{net}}(t) < 0} \big[-P_{\text{net}}(t)\big] \times 1000$$

即仅在**净功率为负的时段**求和：风光不足的差额全部由电网补足。

---

### 3.4 上网电量 $E_{\text{sell}}$

$$\boxed{E_{\text{sell}} = \sum_{t=0}^{23} P_{\text{sell}}(t) \times 1000 = \sum_{t=0}^{23} \max\big(P_{\text{net}}(t),\ 0\big) \times 1000 \quad (\text{kWh})}$$

**代码对应**（第60行）：

```python
sell_kwh = float(np.sum(sell)) * 1000
```

等价写法：

$$E_{\text{sell}} = \sum_{t: P_{\text{net}}(t) > 0} P_{\text{net}}(t) \times 1000$$

仅在**净功率为正的时段**求和：风光盈余全部上网。

---

## 四、四个量之间的恒等关系验证

由功率平衡逐时成立，累加后可得宏观能量守恒：

$$\boxed{E_{\text{RE}} + E_{\text{buy}} = E_{\text{sell}} + E_{\text{load}}^{\text{total}}}$$

验证（带符号）：

$$\underbrace{\sum_t (P_{\text{wind}} + P_{\text{solar}}) \cdot 1000}_{E_{\text{RE}}} + \underbrace{\sum_t P_{\text{buy}} \cdot 1000}_{E_{\text{buy}}} = \underbrace{\sum_t P_{\text{sell}} \cdot 1000}_{E_{\text{sell}}} + \underbrace{\sum_t (P_{\text{eq}} + P_{\text{load}}) \cdot 1000}_{E_{\text{load}}^{\text{total}}}$$

这个恒等式是代码中 `balance_ok` 校验的理论基础（第115–122行，容差 $10^{-4}$ MW）。

## 五、汇总表

| 指标 | 符号 | 计算式 | Q1典型值 |
|------|:--:|------|:--:|
| 新能源发电量 | $E_{\text{RE}}$ | $\sum_t (P_{\text{wind}} + P_{\text{solar}}) \times 1000$ | **761,760 kWh** |
| 日用电量 | $E_{\text{load}}^{\text{total}}$ | $\sum_t (P_{\text{eq}} + P_{\text{load}}) \times 1000$ | **~498,000 kWh** |
| 网购电量 | $E_{\text{buy}}$ | $\sum_t \max(-P_{\text{net}}, 0) \times 1000$ | 由净功率负时段决定 |
| 上网电量 | $E_{\text{sell}}$ | $\sum_t \max(P_{\text{net}}, 0) \times 1000$ | 由净功率正时段决定 |

其中 $P_{\text{net}}(t) = P_{\text{wind}}(t) + P_{\text{solar}}(t) - 20.75 - P_{\text{load}}(t)$。

---

## 六、理解要点

1. **$P_{\text{eq}} = 20.75$ MW 是常数**（Q1核心假设：设备24h满负荷），因此设备日用电量 = $20.75 \times 24 \times 1000 = 498,000$ kWh 是**固定值**，不随风光变化。

2. **购电和售电是"被动导出"的**——Q1没有优化自由度，$E_{\text{buy}}$ 和 $E_{\text{sell}}$ 由风光出力与固定负荷的差值完全决定，这正是不达标（$\eta_1 = 28.2\%$）的根源：正午光伏大发被迫上网，夜间风光不足被迫购电。

3. **×1000 的物理意义**：MW × 1h = MWh，×1000 = kWh。竞赛论文中建议统一用 MWh 书写以简洁，但代码用 kWh 是为了与电价单位（元/kWh）对齐，避免后续成本计算时出错。
